# 入职智引系统串行开发任务看板

## 全局环境基线（全员必读）

- Python 版本: 3.12.10 (严禁使用其他版本，避免依赖冲突)
- LLM 模型: qwen3.7-plus (通过百炼 API 调用)
- 代码风格：禁止使用emoji
- Embedding 模型: text-embedding-v4
- 鉴权方式: 统一读取系统环境变量 API_KEY，代码中禁止硬编码
- 核心框架: LangChain + Pandas + NumPy + ChromaDB + FastAPI + Vue3

## 项目文件结构

```
OnboardingBot/
├── backend/
│   ├── data/raw/          # 原始 PDF
│   ├── data/rules/        # CSV 规则表
│   ├── VectorStore/       # 向量库
│   ├── src/tools/         # 计算函数
│   ├── src/core/          # LangChain 链路
│   ├── src/api/           # FastAPI 接口
│   └── tests/             # 测试用例
├── frontend/              # Vue3 前端
└── docs/                  # 文档
```

## 阶段一：成员二（数据工程师）—— 知识资产与计算引擎构建

阶段目标：产出可直接被后端加载的向量库文件、结构化规则表及经过单元测试的计算函数。
前置条件：无（项目起点）

- 1. 工程基座与环境验证

  - 执行动作：初始化 Git 仓库；配置 .gitignore。
  - 验收标准：Git 仓库结构规范，.gitignore 配置正确。

- 2. PDF 知识库构建（LangChain + ChromaDB）

  - 执行动作：使用 PyPDFLoader 读取《员工手册》；清洗噪音后使用 RecursiveCharacterTextSplitter (chunk_size=512, overlap=50) 切片；使用 DashScopeEmbeddings(model="text-embedding-v4") 向量化；写入本地 ./VectorStore/ 并确保持久化生效。
  - 验收标准：./VectorStore/ 目录生成，重启后能成功加载，检索测试 Top-3 准确率达标。

- 3. 结构化数据提取与计算引擎（Pandas + NumPy）

  - 执行动作：用 pdfplumber 提取薪资/绩效表格，Pandas 清洗导出为 CSV；编写纯函数 calculate_deduction(salary, late_count) 等，内部使用 Pandas/NumPy 运算；编写 pytest 单元测试覆盖边界值。
  - 验收标准：CSV 文件无缺失；pytest 全部通过；计算函数签名清晰、文档完整，可供组长直接 import。

- 4. 阶段一交付物打包与交接

  - 执行动作：整理交付清单：① ./VectorStore/ 目录 ② data/rules/*.csv ③ src/tools/calculator.py ④ 检索质量测试报告；与组长进行面对面交接演示。
  - 交接验收标准：组长能在自己的环境中成功加载向量库、读取 CSV、调用计算函数返回正确结果。签字确认后方可进入阶段二。

## 阶段二：组长（后端开发）—— RAG 引擎与 API 服务化

阶段目标：基于阶段一交付物，构建支持多轮对话、流式输出、Function Call 的完整后端服务。
前置条件：阶段一交接验收通过

- 1. LangChain RAG 链路搭建

  - 执行动作：加载成员二交付的 ./VectorStore/；初始化 ChatTongyi(model="qwen3.7-plus")；使用 LCEL 构建检索链；编写 HR 助手 Prompt Template（含引用溯源指令）；集成 RunnableWithMessageHistory 实现多轮对话记忆（本地 JSON/SQLite 存储）。
  - 验收标准：终端测试多轮追问，上下文关联正确，引用来源标记格式统一。

- 2. Function Call 集成与异常处理

  - 执行动作：导入成员二的 calculator.py，用 @tool 封装并 bind_tools 到 LLM；实现 API 超时重试、限流降级、输入校验（防 Prompt 注入）；所有关键操作接入 logging 日志。
  - 验收标准："迟到3次扣多少钱"等计算类问题精准回答；恶意输入被拦截；API 异常时返回友好提示而非 500。

- 3. FastAPI 流式接口封装

  - 执行动作：使用 astream_events + StreamingResponse 实现 SSE 流式输出；封装 POST /api/chat、GET /api/health；配置 CORS；输出完整的接口文档（JSON Schema），包含请求/响应示例、字段说明、错误码定义。
  - 验收标准：Postman/Swagger 测试流式响应正常；接口文档已交付给成员三。

- 4. 阶段二交付物打包与交接

  - 执行动作：整理交付清单：① 可运行的 FastAPI 服务 ② 完整接口文档（JSON Schema）③ Mock 数据生成脚本 ④ 后端自测报告；与成员三进行接口联调预演。
  - 交接验收标准：成员三能用 Mock 数据跑通前端完整流程；接口文档字段与后端实际返回一致。签字确认后方可进入阶段三。

## 阶段三：成员三（前端开发）—— 交互实现与体验打磨

阶段目标：基于阶段二接口文档，构建流畅、可信、响应式的对话界面。
前置条件：阶段二交接验收通过

- 1. Mock 驱动的前端基座搭建

  - 执行动作：Vue3 + Vite 初始化；根据组长交付的 JSON Schema 生成 Mock 数据；完成聊天界面静态布局、Markdown 渲染组件集成、前端代理配置。
  - 验收标准：使用 Mock 数据可完整展示对话流程，无需等待后端真实接口。

- 2. SSE 流式对接与多轮会话管理

  - 执行动作：对接真实 /api/chat SSE 接口，实现打字机效果；实现 session_id 管理与 localStorage 历史存储；实现快捷提问按钮自动发送。
  - 验收标准：流式显示平滑无闪烁；刷新页面历史对话保留；多轮对话上下文正确传递。

- 3. 引用溯源高亮与体验优化

  - 执行动作：解析引用标记，实现点击弹窗/侧边栏高亮原文；增加 AI 思考动画、网络超时/断线重连提示；适配移动端/PC端响应式布局。
  - 验收标准：引用溯源交互流畅；异常场景有友好提示；多设备布局正常。

- 4. 阶段三交付物打包与交接

  - 执行动作：整理交付清单：① 可构建的前端工程 ② 前端自测报告（含兼容性测试）③ 演示环境部署脚本；与成员四进行测试验收。
  - 交接验收标准：成员四能独立启动前后端完整系统；核心功能无阻塞性 Bug。签字确认后方可进入阶段四。

## 阶段四：成员四（文档与项目管理）—— 质量保障与价值提炼

阶段目标：全面验证系统质量，提炼技术亮点，产出高质量答辩材料。
前置条件：阶段三交接验收通过

- 1. 全链路系统测试

  - 执行动作：执行测试用例表（入职/考勤/离职/多轮追问/边界问题/安全测试）；重点验证：防幻觉能力、Function Call 精度、多轮上下文、异常处理；记录所有缺陷并推动修复。
  - 验收标准：输出《系统测试报告》，所有 P0/P1 级缺陷已修复闭环。

- 2. 架构文档与亮点提炼

  - 执行动作：绘制最终系统架构图（标注 LangChain/Qwen3.7/text-embedding-v4/Pandas/ChromaDB 各组件位置）；提炼技术亮点（串行协作模式、数据本地化、结构化计算引擎、多轮记忆等）。
  - 验收标准：架构图准确反映实际实现；亮点总结 200 字以内，突出差异化价值。

- 3. 答辩 PPT 与演示脚本

  - 执行动作：制作答辩 PPT（痛点→架构→亮点→Demo→总结）；编写演示脚本（含正常流程 + 刁钻问题 + 异常恢复展示）；组织至少 2 次全员彩排。
  - 验收标准：PPT 逻辑清晰；Demo 演示全程无故障；彩排反馈问题已全部解决。

- 4. 最终交付物归档

  - 执行动作：整合所有文档（计划书、测试报告、架构图、PPT、README）；打 Tag 发布 Release；提交实训总结报告。
  - 最终验收标准：所有交付物齐全、格式规范、可追溯；项目可一键启动演示。

## 串行模式风险管控要点

| 风险点               | 应对措施                                                     |
| -------------------- | ------------------------------------------------------------ |
| 上游延期导致下游阻塞 | 每个阶段设置 80% 检查点，提前预警；下游可在上游未完全交付时先做技术预研 |
| 交接物质量不达标     | 严格执行签字确认制，不合格不打回重做，不计入下游工时         |
| 环境不一致           | 阶段一即锁定 requirements.txt + Python 3.12.10，后续阶段禁止私自升级依赖 |
| 接口变更导致前端返工 | 阶段二交付接口文档后，任何变更需书面通知成员三并同步更新 Mock 数据 |
| 演示环境崩溃         | 阶段三交付部署脚本，阶段四每次彩排前重置环境验证一键启动可靠性 |
